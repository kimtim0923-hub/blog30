## 2. 블로그 콘텐츠 시트

blog_generator.py가 생성한 블로그 글을 저장한다. 시트가 없으면 자동 생성된다.

| 열 | 컬럼명 | 데이터 타입 | 예시값 | 비고 |
|----|--------|-----------|--------|------|
| A | 툴이름 | 문자열 | `Sora` | 툴 DB의 A열과 동일. 검색 키 |
| B | 제목 | 문자열 | `Sora 리뷰: AI 영상 생성, 실제로 써보니 이랬습니다 (2026)` | H1 제목. Claude가 생성 |
| C | 내용 | 문자열 (장문) | `> 이 글에는 제휴 링크가...` | 블로그 본문 전체. 마크다운 형식 |
| D | 생성일시 | 날짜 문자열 | `2026-03-10 14:30` | `YYYY-MM-DD HH:MM` 형식 |
| E | 업로드상태 | 문자열 | `글완성`, `업로드완료` | tistory_uploader.py가 업데이트 |
| F | 글유형 | 문자열 (1자) | `B`, `C`, `D`, `E` | 아래 유형 참조 |

### F열 글유형 정의

| 유형 | 이름 | 설명 |
|------|------|------|
| `B` | 단일 툴 리뷰 | 한 툴에 대한 심층 리뷰 (2,500~3,000자) |
| `C` | 두 툴 비교 | 같은 카테고리 두 툴 비교 (2,000~2,500자) |
| `D` | 대안 소개 | 유명 툴의 대안 TOP N (2,500~3,000자) |
| `E` | 순위/리스트 | 카테고리별 TOP N 순위 (3,000~3,500자) |

### E열 상태값 정의

| 상태 | 의미 |
|------|------|
| `글완성` | 생성 완료, 업로드 대기 (tistory_uploader.py 대상) |
| `업로드완료` | 티스토리 발행 완료 |
| `업로드오류` | 업로드 중 오류 발생 |

## 시트 간 관계

```
툴 DB (A: 툴이름)
  │
  ├──▶ 블로그 콘텐츠 (A: 툴이름)
  │      blog_generator.py가 생성, tistory_uploader.py가 읽음
  │
  └──▶ 주간 로그 (C: 툴이름)
         매주 수집 기록 누적
```

- **툴 DB**는 마스터 테이블. 모든 툴의 최신 정보를 유지한다.
- **블로그 콘텐츠**는 글 생성 결과 저장소. 툴이름으로 툴 DB와 연결된다.
- **주간 로그**는 히스토리 테이블. 같은 툴이 여러 주에 걸쳐 등장할 수 있다.
